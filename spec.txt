1. System Overview
This document specifies the steering control algorithm for a 4-wheel Skid-Steer robot. Its
purpose is to translate operator joystick commands into correct per-motor CAN throttle
commands, while preventing rollover and providing smooth motion.
•​

Configuration: 4-wheel Skid-Steer, two motor controllers (Left / Right)

•​

CAN protocol: DriveCommand at 0x300+NodeID, SpeedTelemetry at 0x310+NodeID

•​

Operating mode: Speed Mode (MotorMode = 01 in DriveCommand)

•​

Guiding principle: Operator-requested linear speed is sacred. Turn radius is adapted
to prevent rollover. Only in a physical impossibility is speed reduced to honour the
requested turn angle.

3. Unit Conversions
RPM → m/s
V [m/s] = RPM × 2π × r / 60

m/s → CAN Throttle
Throttle = round( (V / V_max) × 1000 )
Throttle = clamp(Throttle, −1000, +1000)

CAN accepts integer throttle in the range [−1000, +1000]. The final clamp is a numerical
safety guard only — if the algorithm is correct the value should already be within bounds.

V_for_omega is the reference used for anti-rollover calculations in Stage 2. Taking the
maximum of actual and commanded speed ensures the turn limit is computed conservatively
when the robot is already moving faster than the current command.

Stage 2 — TurnAmount by Speed Zone
Three zones based on V_for_omega:

Zone A — Pivot (V_for_omega ≤ V_pivot)
At near-zero speed there is no centrifugal force. TurnAmount equals maximum differential
directly:
TurnAmount = SteerCmd × V_max

Zone B — Dynamic (V_for_omega ≥ V_blend_end)
Anti-rollover constraint: lateral acceleration must not exceed a_lat_max.
ω_max
= a_lat_max / V_for_omega
TurnAmount = SteerCmd × (ω_max × W / 2)

Derivation: a_lat = V × ω ≤ a_lat_max ⇒ ω ≤ a_lat_max / V. The wheel speed difference
equals ω × W, so each side contributes ω × W/2.

Zone C — Blend (V_pivot < V_for_omega < V_blend_end)
Linear interpolation between pivot and dynamic results:
ratio
= (V_for_omega − V_pivot) / (V_blend_end − V_pivot)
// ∈ (0, 1)
TurnAmount_pivot
= SteerCmd × V_max
ω_max_blend
= a_lat_max / V_blend_end
TurnAmount_dynamic = SteerCmd × (ω_max_blend × W / 2)
TurnAmount
= (1 − ratio) × TurnAmount_pivot + ratio ×
TurnAmount_dynamic

Stage 3 — Conflict Resolution
Principle: preserve the turn, sacrifice linear speed.
peak_side = |V_target_drive| + |TurnAmount|
if peak_side > V_max:
V_target_drive = sign(V_target_drive) × (V_max − |TurnAmount|)

After this step, both motor targets computed in Stage 4 are guaranteed to lie within [−V_max,
+V_max].

Stage 4 — Per-Motor Targets
Target_left = V_target_drive + TurnAmount
Target_right = V_target_drive − TurnAmount

Sign convention: SteerCmd > 0 is a right turn. Left wheel accelerates, right wheel
decelerates (or reverses in pivot).

Stage 5 — Slew Rate Limiter
Prevents instantaneous torque jumps. Applied independently to each side.
max_delta

= MaxAccel × dt

delta_left = clamp(Target_left − Cmd_left_prev,
+max_delta)
Cmd_left
= Cmd_left_prev + delta_left

−max_delta,

delta_right = clamp(Target_right − Cmd_right_prev, −max_delta,
+max_delta)
Cmd_right
= Cmd_right_prev + delta_right

Cmd_left and Cmd_right become the new previous values for the next cycle.

Stage 6 — CAN Throttle Output
Throttle_left = round( (Cmd_left / V_max) × 1000 )
Throttle_right = round( (Cmd_right / V_max) × 1000 )

Send via DriveCommand frame (CAN ID 0x300 + NodeID) with MotorMode = 01 (Speed
Mode).

5. Safeguards
5.1 Telemetry Timeout
If SpeedTelemetry has not been received from either side within telem_timeout seconds:
•​

Set DriveCmd = 0 and SteerCmd = 0 for this cycle

•​

Log / raise a fault flag

•​

Slew Rate Limiter remains active — the robot decelerates smoothly to zero

5.2 Initialisation
Cmd_left_prev and Cmd_right_prev must be initialised to the first telemetry reading
received, not to 0. Initialising to 0 when the robot is already moving causes a large initial
delta that the slew limiter cannot suppress in one cycle.

5.3 Final Output Clamp
Throttle_left = clamp(Throttle_left, −1000, +1000)
Throttle_right = clamp(Throttle_right, −1000, +1000)

This is purely a numerical safety guard. Under normal operation the values should be in
range before clamping.

5.4 Joystick Dead-Band (optional)
If |DriveCmd| < deadzone, treat DriveCmd as 0. If |SteerCmd| < deadzone, treat SteerCmd
as 0. This eliminates stick noise at rest.

6. Out of Scope
The following are not part of this algorithm. Implement them in a separate layer:
•​

Loop timing and scheduling

•​

•​
